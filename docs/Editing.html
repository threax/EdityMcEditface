Edity McEditface Manual

Editing

Edity McEditface tries to give a natural editing experience that is similar to actually visiting the website itself. This means that the wysiwyg is pretty much 100% accurate since you edit the pages directly.

Creating a New Page

To create a new page all that is required is visiting it in your browser. Instead of a 404 error you will be shown a screen of templates to choose from to create your new page. The easiest way to create a new page is to create a link for it and then visit the link.

Editing a Page

Basics

Once you get to the edit screen for a page, click on the text, a toolbar with some formatting options should pop up and you can begin typing in your site content. Your changes are automatically saved to the server. You can also paste images directly from the clipboard into the page and the files will be automatically uploaded, named and linked.

History

This will show the git history of the page you are currently editing.

Media

Upload files directly to the cms.

Edit Source

Edit the page source directly. Don't spend too much time formatting since CKEditor will probably wipe out your pretty formatting.

Publish

Publish the website by compiling it based on your current compiler settings from your configuration.

Settings

Change the page settings, set the title or anything else needed. You must reload the page after applying these to see the effect.

Sync

Sync your git changes with others. This will push and pull from the remote repositories as needed.

Commit

Commit your current changes to git.

Save

Save the current page content. This is done automatically 5 seconds after you finish editing something.

Preview

See what the page will look like outside the editor. Not usually needed since you are seeing that anyway, but this is an easy way to remove the editor from a page and look at it. Clicking any links from the preview will take you to the editor for that page.
